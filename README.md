# SocialFlow - 代办-经办-查验关系网状图生成器

该程序由本人2024年写的第一版代办人经办人模型修改而来，现在三个关系关联。根据业务数据（Excel格式）自动生成**分部门**的经办人、代办人、查验员三方关系网络图，以代办人为中心，标注业务互动次数，高亮部门内关系最密切的连接。

## 功能特性

- **以代办人为核心**：只构建「代办人 ↔ 经办人」和「代办人 ↔ 查验员」两种边，避免经办人与查验员直接相连
- **动态权重**：同一对人员共同出现一次，权重 +1；仅显示权重 >2 的关系
- **复合角色着色**：根据人员兼任情况使用7种不同颜色区分  
  （纯经办人、纯代办人、纯查验员、经办+代办、经办+查验、代办+查验、三者皆是）
- **部门独立子图**：每个业务部门单独绘制一个网络子图，排列在一张大图上
- **最密切关系标红**：每个子图中权重最高的边自动加粗并显示为红色
- **边权重标注**：每条边上直接显示具体的业务互动次数（3、4、5…）

## 逻辑说明

1. 读取 Excel，要求包含以下列：
   - `经办人`、`代办人`、`查验员姓名`、`业务部门`
2. 按部门分组，对每个部门：
   - 遍历每一行，以 `代办人` 为起点，分别与 `经办人` 和 `查验员姓名` 建立边，累加权重
   - 过滤掉权重 ≤2 的边
   - 移除孤立节点
3. 识别每个人员所属的角色集合（经办人集、代办人集、查验员集），分配颜色
4. 计算最大权重，将该部门内权重最高的唯一一条边标记为红色，其余为黑色
5. 使用弹簧布局（spring layout）绘制网络，并添加边权重标签、图例、部门标题
6. 将所有子图整合为一张大图保存为 `combined_network.png`

## 安装与运行

### 环境要求
- Python 3.11.5
- 建议使用虚拟环境

### 步骤

1. 克隆或下载本仓库
2. 打开终端，进入项目目录
   ```bash
   cd SocialFlow
   ```
3. 创建虚拟环境（可选但推荐）
   ```bash
   python -m venv venv
   ```
4. 激活虚拟环境
   - Windows: `.\venv\Scripts\activate`
   - macOS/Linux: `source venv/bin/activate`
5. 安装依赖
   ```bash
   pip install -r requirements.txt
   ```
   > `requirements.txt` 内容：`pandas`, `networkx`, `matplotlib`, `openpyxl`
6. 准备数据文件  
   将待分析的 Excel 文件命名为 **`代办情况.xlsx`**，放在项目根目录，确保包含上述必需的列名。
7. 运行脚本
   ```bash
   python combine_networks.py
   ```
8. 查看结果  
   生成的 `combined_network.png` 将出现在当前目录中。

### 自定义阈值
如需修改权重阈值（默认为 `>2`），编辑 `combine_networks.py` 中的 `min_weight=2` 参数。

## 示例输出

- 每张子图左上角显示部门名称及节点/边数量
- 图例说明所有角色颜色及红线含义
- 文件分辨率 300 DPI，适合直接用于报告或展示
